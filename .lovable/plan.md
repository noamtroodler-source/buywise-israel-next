

# Add 5 Mock Nearby Sold Transactions for German Colony Listing

## Problem
The "Recent Nearby Sales" section on the German Colony property page currently shows only 1 comp. The `get_nearby_sold_comps` database function filters by room count (property bedrooms +/- 1, so 7-9 rooms for this 8BR house) AND a 500m radius. The previously inserted mock data either had too few rooms or coordinates outside the search radius.

## Solution
Insert 5 new `sold_transactions` records into the database with:
- **Rooms**: 7, 8, or 9 (to match the filter range for an 8BR property)
- **Coordinates**: All within ~100-400m of the property (31.7615, 35.2210) to fall within the 500m Haversine search
- **City**: Jerusalem
- **Property type**: `house` (luxury homes in German Colony)
- **Prices**: Anchored to the listing's price/sqm (~$25,875/sqm) with a +/-15% variance, ranging from roughly $4M to $9M depending on size
- **Source**: `mock_seed` for idempotent management
- **Dates**: Spread across the last 12 months for a natural timeline
